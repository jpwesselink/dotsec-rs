    #Getting Started

    dotsec encrypts your .env files into .sec files. The .sec file is committed to git as the single source of truth for your secrets — no more sharing credentials over Slack or managing secret vaults.

    Encryption uses age (X25519 + ChaCha20-Poly1305) by default. AWS KMS is also supported for enterprise teams.

    #Install

    npm install -g dotsec

    Distribution is npm-only — the package bundles a native per-platform binary. Prefer a dev-dependency or one-shot npx? See Setup → Install.

    #Your first secret

    dotsec set API_KEY sk-live-xxx --encrypt  # add a secret (auto-creates .sec + keypair)
dotsec set PORT 3000                      # add a plaintext var
dotsec run -- node server.js              # run with decrypted env vars

    That's it. Three things just happened:

    1. .sec was created — encrypted, commit this
    2. .sec.key was created — your private key, auto-added to .gitignore, never commit this
    3. node server.js ran with API_KEY and PORT in its environment, decrypted in memory only

    Already have a .env? Import it:

    dotsec import      # interactive
dotsec import -y   # auto-detect types and encryption

    #How it works

    .env (plaintext, gitignored)          .sec (encrypted, committed)
┌─────────────────────────┐           ┌────────────────────────────┐
│ DATABASE_URL=postgres://│  encrypt  │ # @dotsec(format=v3,       │
│ API_KEY=sk-live-xxx     │ ───────▶  │ #   mac=..., dek=...)      │
│ DEBUG=true              │           │ DATABASE_URL=ENC[base64]   │
└─────────────────────────┘           │ API_KEY=ENC[base64]        │
                                 ◀─── │ DEBUG=true                 │
                                decrypt└───────────────────────────┘

    Each secret is encrypted individually with AES-256-GCM using a data encryption key (DEK). The DEK is wrapped by your keypair (or AWS KMS) and stored in the @dotsec(...) directive. This makes .sec files git-mergeable — changing one secret only affects that line.

    #Everyday commands

    dotsec set KEY value              # add or update a variable
dotsec show                       # display decrypted .sec contents (masked)
dotsec run -- npm start           # inject decrypted vars into a command
dotsec export -o .env             # .sec → .env (decrypts to file)
dotsec validate                   # check directives and values
dotsec diff .sec.staging          # compare .sec files
dotsec rotate-key                 # generate new DEK, re-encrypt all values

    Full reference: Commands.

    #Team sharing

    Share the .sec.key file with teammates over a secure channel (a password manager, not Slack). For CI, set DOTSEC_PRIVATE_KEY as a secret — see CI/CD for copy-paste GitHub Actions and GitLab recipes.
